working directory : .git 존재하는 폴더
-> 해당 폴더에서 변경되는 모든 내용을 추적

staging(stage) area : 스테이지에 올라간 파일 영역
-> 기록하려는 파일이 머무는 곳
(이 곳에 올라온 파일들을 기록할 예정이다)

commit : stagging area에 올라온 파일을 local repository에 저장(맡기는) 명령어

local(현재 컴퓨터) repository(저장소) : 내 pc에 변경 내용을 저장할 저장소

remote(원격) depository(저장소) : 

집에서 학원에서 수업한 내용(github에 push된 내용 얻어오기
1. 수업한 내용을 저장한 폴더를 생성

2. github 로그인

3. 원격 저장소(remote repository) 페이지로 이동

4. Code 버튼 클릭 -> clone(복제) -> 주소 복사 

5. 소스트리 -> 새 탭 -> Clone 버튼 클릭

6. 소스 경로 입력 창에 복사한 주소 붙여넣기

7. 두 번째 줄 탐색을 클릭해서 복제한 저장소의 위치를 지정

8. 선택한 위치 뒤에 \폴더명을 추가 작성

9. 클론 버튼 클릭
